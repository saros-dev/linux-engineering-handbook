# Contributing

Thank you for contributing to Linux Engineering Handbook.

## Getting Started

1. Fork the repository
2. Create a branch
3. Make your changes
4. Commit with a meaningful message
5. Open a Pull Request

## Content Standards

Every chapter should include:

* Overview
* Why It Matters
* Core Concepts
* Diagrams
* Real-World Examples
* Interview Questions
* Summary

## Commit Messages

Examples:

* Add Linux Architecture chapter
* Improve VFS explanation
* Fix typo in Linux History

Avoid:

* update
* fix
* changes

## Pull Requests

Before submitting:

* Verify Markdown formatting
* Check spelling and grammar
* Test code examples
* Verify internal links
